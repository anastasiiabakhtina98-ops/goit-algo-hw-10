# Обчислення визначеного інтеграла методом Монте-Карло

## Постановка задачі

Потрібно обчислити визначений інтеграл функції  яка задає червону криву на графіку, а площа під нею на відрізку \([0, 2]\) відповідає сірій заштрихованій області. 

![Опис картинки](images/graphic.png)

Інтеграл обчислюється трьома способами:

- методом Монте‑Карло;
- чисельним методом через функцію `scipy.integrate.quad`;
- аналітично за формулою первісної.

## Результати обчислень

Отримані числові значення:

- Оцінка інтеграла методом Монте‑Карло: **2.6836394452530574**
- Інтеграл `quad`: **2.6666666666666665**, оцінка похибки `quad`: **2.9605947323337504e-14**
- Аналітичний результат: **2.6666666666666665**  
- Різниця Monte Carlo – `quad`: **0.016972778586390902**
- Різниця Monte Carlo – аналітичний: **0.016972778586390902**.

## Аналіз точності

1. Значення, отримане за допомогою `quad`, практично співпадає з аналітичним результатом (різниця лише на рівні числової похибки подвійної точності ~10⁻¹⁴), тому його можна вважати еталонним.
2. Метод Монте‑Карло дав значення 2.6836, що відрізняється від еталону приблизно на 0.017, тобто на ~0.64% від істинного значення інтеграла. 
3. Для класичного Монте‑Карло така похибка є очікуваною, оскільки точність зростає як \(1/sqrt{N}\), де \(N\) — кількість випадкових точок; збільшення \(N\) зменшить різницю між результатами.


## Висновки

- Аналітичне обчислення та результат функції `quad` практично збігаються, що підтверджує коректність як математичної формули інтеграла, так і роботи чисельного методу `quad` для даної простої функції.  
- Оцінка інтеграла методом Монте‑Карло близька до еталонного значення, але має невелику випадкову похибку, що є природною властивістю стохастичного методу.  
- Отримані результати демонструють, що реалізований алгоритм Монте‑Карло працює правильно: його наближений результат знаходиться в радіусі точного значення і збігався б до нього ще краще при збільшенні кількості випадкових точок \(N\).
